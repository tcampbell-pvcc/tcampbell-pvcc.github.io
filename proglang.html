Chapter 6: Programming Languages

Term Definition
Machine language: sequences of instructions that are encoded as numeric digits
Debugging: locating and correcting errors in written programs
Assembler: developed to convert mnemonic expressions into machine language instructions
Assembly language: a mnemonic system for representing programs
Translator: translates programs expressed in high-level primitives into machine-language programs
Compiler: translation programs
Interpreter: another means of implementing third-generation languages
Cross-platform software: software that is independent of an operating system's design as well as the machine's hardware design and is therefore executable throughout a network
Natural language: English, German, Latin, etc.- evolved over time without formal grammatical analysis
Formal language: programming languages- defined by grammars
Machine independence: does not rely on the characteristics of a particular machine
High-level programming language: provide more abstractions for the programmer and make it easier for people to read and write a program
Object-oriented programming (OOP): a software system is viewed as a collectio of units, called objects, each of which is capable of performing the actions that are immediately related to itself as well as requesting actions of other objects
Scripting language: typically used to perform administrative tasks rather than to decelop complex programs
Data types:
Integer: refers to numeric data consisting of whole numbers
Float: refers to the numeric data that might contain values other that whole numbers
String: sequence of charachters and the most commonly used data type to store text
Primitive data type: data types that are included as primitives in a programming language, such as int for integer and char for character
Data Stricture: the conceptual shape or arrangement of data
Python collection data structures
List: an ordered collection of items
Tuple: a built-in data structure that is an ordered collection of objects
Set: an unordered and unindexed collection of elements
Dictionary: allows us to easily write very efficient code
Function: a set of instructions for performing a task that can be used as an abstract tool by other program units
Local variable: a variable declared within a function, meaning that it can be referenced only within that function
Global variable: variables whose scopes are not restricted to a particular part of a program
Source program: the program in its original form
Object program: the translated version of the program
